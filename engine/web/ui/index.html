백업 대상 · 제외 규칙

백업 루트는 /home · /var/lib/docker/volumes 로 고정입니다(컨테이너 마운트 기반). 그 안에서 무엇을 제외할지를 아래에서 정합니다. 한 줄에 하나, 변경은 다음 백업부터 적용됩니다.

목적지 추가 (간편)

WebDAV · Synology · SFTP · FTP · S3 같은 목적지를 한글 폼으로 바로 추가합니다. Google Drive 등 OAuth는 아래 "rclone 공식 설정"에서 추가하세요.

이름영문/숫자/_-. (REMOTE_NAME 으로 쓰는 이름)
유형

rclone 공식 설정 · 설정된 목적지

Google Drive 등 OAuth나 세밀한 편집은 rclone 공식 설정 화면(이 창 안 모달, 영어)에서 합니다. 추가/변경한 remote는 즉시 사용 가능(엔진 재시작 불필요), 활성 목적지 변경은 .env의 REMOTE_NAME 수정 후 재시작. 별도 포트 노출 없이 대시보드를 통해 안전하게 연결됩니다.

설정된 목적지

rclone CLI (명령 실행)

한 줄(one-shot) 실행입니다. 조회: listremotes · lsd gdrive: · about gdrive:. 설정은 대화형 마법사 대신 한 줄 명령으로: config create 이름 webdav url=… user=… pass=… · config update · config delete 이름 · config show · config redacted. 대화형 config(단계 선택식)·삭제/전송류는 불가 — 그건 위 "목적지 추가" 폼이나 "rclone 설정 열기"(GUI)를 쓰세요.
