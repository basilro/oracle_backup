설정

백업 대상 · 제외 규칙

백업 루트는 /home · /var/lib/docker/volumes 로 고정입니다(컨테이너 마운트 기반). 그 안에서 무엇을 제외할지를 아래에서 정합니다. 한 줄에 하나, 변경은 다음 백업부터 적용됩니다.

백업 대상 경로 · 데이터베이스

백업할 소스 경로입니다. /var/lib/docker/volumes는 항상 포함됩니다. 여러 경로는 공백으로 구분하세요.

소스 경로기본 /home
데이터베이스 덤프 작업 기본값(미저장)

덤프할 DB를 추가/삭제합니다. 컨테이너가 없거나 멈춰 있으면 자동으로 건너뜁니다. 데이터 경로는 덤프 성공 시 원본 이중 백업을 막기 위해 제외됩니다(비우면 제외 안 함). 인증정보는 secrets/db-creds.env에서 관리합니다. 모두 삭제하면 DB 덤프를 건너뜁니다.

알림 (Discord/Slack webhook)

백업 성공/실패 시 이 webhook으로 알림을 보냅니다. Discord·Slack 호환. 비우고 저장하면 알림이 꺼집니다. (/secrets/discord-webhook 파일이 있으면 그게 우선합니다.)

Webhook URL

목적지 추가 (간편)

WebDAV · Synology · SFTP · FTP · S3 같은 목적지를 한글 폼으로 바로 추가합니다. Google Drive 등 OAuth는 아래 "rclone 공식 설정"에서 추가하세요.

이름영문/숫자/_-. (REMOTE_NAME 으로 쓰는 이름)
유형

rclone 공식 설정 · 설정된 목적지

Google Drive 등 OAuth나 세밀한 편집은 rclone 공식 설정 화면(이 창 안 모달, 영어)에서 합니다. 추가/변경한 remote는 즉시 사용 가능(엔진 재시작 불필요), 활성 목적지(원격) 변경은 아래 "백업 저장 위치" 카드의 원격 전환으로(재시작 불필요). 별도 포트 노출 없이 대시보드를 통해 안전하게 연결됩니다.

설정된 목적지

백업 저장 위치 (원격 경로)

현재 원격(—) 안에서 저장소가 위치한 경로입니다. 변경하면 기존 백업 데이터를 새 경로로 이동(복사→검증→전환→원본 삭제)합니다. 원격이 연결된 상태에서만 변경할 수 있습니다.

rclone CLI (명령 실행)

한 줄(one-shot) 실행입니다. 조회: listremotes · lsd gdrive: · about gdrive:. 설정은 대화형 마법사 대신 한 줄 명령으로: config create 이름 webdav url=… user=… pass=… · config update · config delete 이름 · config show · config redacted. 대화형 config(단계 선택식)·삭제/전송류는 불가 — 그건 위 "목적지 추가" 폼이나 "rclone 설정 열기"(GUI)를 쓰세요.
